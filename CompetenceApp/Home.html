Competence Management System

A comprehensive solution for managing student tests and generating detailed reports.

About the Project

The Competence Management System is designed to simplify the management of student test data. It allows educators to create forms, track test results, and generate detailed PDF reportsâ€”all from a user-friendly web interface integrated with Google Sheets and Drive.

Functional Specifications

  • Dynamic forms for test data entry.
  • Integrated Google Sheets for real-time data management.
  • Automatic PDF report generation for student performance.
  • Role-based access to ensure data security.

Technical Specifications

This project leverages the following technologies and tools:

  • Languages: JavaScript, HTML, CSS
  • Frontend Framework: Bootstrap
  • Backend: Google Apps Script
  • APIs: Google Sheets API, Google Drive API
  • Version Control: GitHub

Project Demo

The Competence Management System is designed to streamline student test management, report creation, and data storage. Below are some of the core features, demonstrated with screenshots.

Configuration Menu

The web application is a multi-page system, with the Configuration Menu allowing users to select the spreadsheet used to store data, including student information, teacher preferences, and test results. By default, a spreadsheet containing dummy data is presented. Users can also view the history of previously selected spreadsheets. All app-related information is stored in AppData, which is managed either by the deployment owner or the executing user, depending on deployment settings.

Configuration Menu: Select and manage the active spreadsheet.

Test Result Form

This form allows users to input the results of student tests directly into the system. The data is dynamically stored in the selected spreadsheet, ensuring real-time updates and ease of management.

Test Result Form: Input and manage student test results.

Report Creation

Using the Make Report Form, users can select a report reference and generate a PDF report. These reports are stored on the server and include detailed visualizations such as a spider graph to showcase test results. Users can view the list of generated reports, select a specific one, and email it directly to the intended recipient.

Report Creation: Generate and view detailed PDF reports.

Report Management: View and email generated reports.

Solution Overview

  1. See Report

    Generate the PDF on the server as a blob, return the base64-encoded blob to the client, and use JavaScript to render the PDF in a browser.

  2. Download PDF

    Reuse the blob returned by the server, and convert the base64-encoded blob into a downloadable link for the user.

  3. Email the Report

    Pass the blob to a server-side function that attaches it to an email and sends it via GmailApp.

Test Type Management

If new test types need to be added to the system, this can be done through the Test Type Form. This ensures that the system remains flexible and adaptable to evolving requirements.

Test Type Management: Add or modify test types for greater flexibility.
